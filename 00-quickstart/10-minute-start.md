# 10-Minute Start

Purpose: start one small AI-assisted PM workflow without building a platform.

## Minute 0-2: Pick One Real Task

Choose one task that will happen this week:

| Task | Good First Output |
|---|---|
| Requirement review | Risk list and unresolved questions |
| PRD critique | Missing states, edge cases, and acceptance gaps |
| Prototype acceptance | Demo checklist and blockers |
| User feedback synthesis | Themes, evidence, and next research questions |
| Hackathon idea | Problem one-pager and demo spine |

## Minute 2-4: Fill Minimal Context

Copy this into your AI workspace:

```text
Project:
User:
Business goal:
Current decision to make:
Known facts:
Open assumptions:
Constraints:
Sensitive data that must not be shared:
```

For a fuller version, use [`../context/project-brief.md`](../context/project-brief.md).

## Minute 4-7: Run One Skill

Pick one:

- [`../skills/requirement-review/SKILL.md`](../skills/requirement-review/SKILL.md)
- [`../skills/prd-critic/SKILL.md`](../skills/prd-critic/SKILL.md)
- [`../skills/prototype-acceptance/SKILL.md`](../skills/prototype-acceptance/SKILL.md)

Ask the AI to follow the skill exactly. If required inputs are missing, the AI must stop and ask rather than inventing.

## Minute 7-9: Add One Human Gate

Write the decision that only a human can make:

| Gate | Owner | Approval Criteria | Stop Condition |
|---|---|---|---|
| Example: decide whether requirement enters next sprint | PM + tech lead | user value, evidence strength, effort, risk | evidence is weak or sensitive data is missing |

## Minute 9-10: Create The Next Loop

Use [`../loops/pm-loop-canvas.md`](../loops/pm-loop-canvas.md) and fill:

```text
Trigger:
Signal:
Evaluator:
Human Gate:
Memory:
Next Action:
```

## Done Means

- You used real project context.
- You produced a decision-support output, not just polished text.
- You identified what the PM must decide.
- You recorded the next feedback loop.
