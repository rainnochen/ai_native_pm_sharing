# User Feedback Loop

## Objective

Turn user feedback into validated product opportunities without overreacting to noise.

## Trigger

Weekly or when feedback volume spikes.

## Signals

- App reviews.
- Customer support tickets.
- User interviews.
- Usage anomalies.
- Sales or operations feedback.

## Procedure

1. Collect feedback with source and date.
2. Cluster themes.
3. Separate evidence from interpretation.
4. Identify high-impact repeated problems.
5. Validate with data or follow-up research.
6. Decide: act, monitor, research, or ignore.

## Evaluator

- Source traceability.
- Theme recurrence.
- Segment relevance.
- Evidence strength.

## Human Gates

PM approves opportunity priority and any user commitment.

## Memory

Log themes, decisions, and validation results.

## Stop Rules

Stop if feedback contains sensitive personal data that cannot be safely processed.

## Example

Theme: "Users distrust AI restaurant ranking when explanation is missing." Next action: prototype explanation variants and test with target users.

## Failure Modes

- Treating loud feedback as representative.
- Ignoring minority but high-severity signals.
- Letting AI remove contradictory evidence.
