# Eval-Driven Demo

Purpose: build demos around test cases instead of only a happy path.

## Eval Cases

| Case | Input | Expected Behavior | Pass / Fail |
|---|---|---|---|
| Happy path |  |  |  |
| Edge case |  |  |  |
| Unsafe request |  |  |  |
| Missing data |  |  |  |

## Procedure

1. Define eval cases before building.
2. Build the smallest demo that passes the cases.
3. Record failures.
4. Update prompt, context, tool contract, or product design based on failure type.

## Human Gate

Human reviewer decides whether eval coverage is enough for the next milestone.
