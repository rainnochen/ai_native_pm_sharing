# HTML Prototype

Purpose: create a lightweight HTML prototype that PMs can open, review, and iterate.

## Recommended Scope

- One screen or one flow.
- Static mock data.
- No production secrets.
- No external CDN required for critical rendering.

## Checklist

| Item | Status |
|---|---|
| Core user path visible |  |
| Empty / error state included |  |
| Main CTA works or is clearly mocked |  |
| Text fits at desktop and mobile widths |  |
| Sensitive data removed |  |

## Human Gate

PM and design owner decide whether the prototype is good enough for user feedback.
