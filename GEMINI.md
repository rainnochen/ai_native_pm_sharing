# Gemini Guidance

When using Gemini with this repository:

1. Use repository files as the primary source of truth.
2. Ground analysis in provided context and mark uncertainty.
3. Do not use private or internal data unless it is explicitly safe to process.
4. Keep outputs concise enough for PMs to copy into work.
5. Record source links in `ATTRIBUTIONS.md` when new public references are used.

Suggested use cases:

- Summarize a project brief.
- Compare two PM loops.
- Review a Hackathon demo spine.
- Generate questions for a PRD review.
