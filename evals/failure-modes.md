# Failure Modes

Common failure modes in PM AI workflows:

| Failure Mode | Symptom | Prevention |
|---|---|---|
| Polished nonsense | Beautiful output, weak evidence | Require source and assumption labels |
| Automation drift | Loop keeps running but no decision improves | Define owner and evaluator |
| Hidden risk | AI suggests action without approval | Add human gates |
| Context overload | Too much irrelevant context | Use project brief and glossary |
| Metric tunnel vision | Optimizes one metric, harms experience | Add guardrails |
| Prompt patching | Every issue fixed by longer prompt | Update harness, tool, or eval |
