# Hackathon Decision Log

Purpose: record tradeoffs so the team can explain what was cut and why.

| Time | Decision | Options | Reason | Owner | Impact |
|---|---|---|---|---|---|
|  |  |  |  |  |  |

## Common Decisions To Record

- Scope cut.
- Model/provider change.
- Mock vs real integration.
- Demo story change.
- Risk accepted.

## Quality Checks

- Record the reason, not just the decision.
- Keep enough detail for pitch Q&A.
