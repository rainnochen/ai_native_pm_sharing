# Demo Backup Plan

Purpose: keep the demo alive when the internet, model, API, deployment, or laptop fails.

| Failure | Primary Backup | Second Backup | Owner |
|---|---|---|---|
| Network fails | local recording | screenshots |  |
| Model API fails | cached responses | mock mode |  |
| Deployment fails | local run | video |  |
| Live demo bug | skip to stable path | narrated recording |  |

## Backup Assets

- 3-minute screen recording.
- Static screenshots.
- Mock dataset.
- Cached model responses.
- Local run instructions.

## Quality Checks

- Backup is tested before submission.
- Pitch can continue without live internet.
- The team knows when to switch to backup.
