# 24h / 36h / 48h Plan

Purpose: timebox hackathon work around evidence.

## 24-Hour Plan

| Time | Goal | Required Evidence |
|---|---|---|
| 0-2h | Pick problem and demo spine | problem one-pager |
| 2-6h | Burn biggest risk | working spike |
| 6-14h | Build vertical slice | clickable / runnable path |
| 14-20h | Add eval and polish | test cases, fallback |
| 20-24h | Freeze and rehearse | final demo script, backup |

## 36-Hour Plan

| Time | Goal |
|---|---|
| 0-6h | Problem, team, risk spike |
| 6-18h | Vertical slice |
| 18-24h | Eval, UX, fallback |
| 24-30h | Pitch, polish, backup |
| 30-36h | Rehearsal and submission |

## 48-Hour Plan

Use the 36-hour plan, then add deeper user validation, better eval cases, and stronger pitch evidence. Do not use extra time to add unrelated features.
