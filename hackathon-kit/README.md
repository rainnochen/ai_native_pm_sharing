# Hackathon Kit

Purpose: help PM-led teams build a real AI demo, not just a pitch deck.

The target output is:

```text
Repo + Demo + Eval + Decision Log + Backup Plan
```

## Core Templates

| Template | Use |
|---|---|
| [`preflight-checklist.md`](preflight-checklist.md) | Prepare accounts, repo, APIs, deployment, and backups |
| [`team-charter.md`](team-charter.md) | Align roles, decision rules, and working rhythm |
| [`problem-one-pager.md`](problem-one-pager.md) | Define user, problem, evidence, scope, and assumptions |
| [`demo-spine.md`](demo-spine.md) | Define the 3-minute and 5-minute demo path |
| [`24h-36h-48h-plan.md`](24h-36h-48h-plan.md) | Timebox work by evidence, not activity |
| [`judging-rubric.md`](judging-rubric.md) | Score problem, demo, AI native quality, impact, and risk |
| [`risk-register.md`](risk-register.md) | Track technical, product, model, compliance, and demo risks |
| [`eval-template.md`](eval-template.md) | Define demo test cases and success criteria |
| [`decision-log.md`](decision-log.md) | Record key tradeoffs |
| [`demo-backup-plan.md`](demo-backup-plan.md) | Prepare for API, network, model, or deployment failure |

## Principle

Problem first, but demo-able. A big idea that cannot produce evidence in 24-48 hours is not a good hackathon scope.
