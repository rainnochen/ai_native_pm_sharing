# Personal PM Harness

Purpose: build a lightweight personal work system for repeatable AI-assisted PM tasks.

## Harness Canvas

| Layer | Questions | Your Setup |
|---|---|---|
| Knowledge | What business context matters? |  |
| Prompt | Which tasks need standard expression? |  |
| Skill | Which repeatable PM skills are needed? |  |
| Tool | Which tools are allowed? |  |
| Workflow | What is the step-by-step flow? |  |
| Output | What artifact must be produced? |  |
| Feedback | How do you evaluate and update? |  |
| Memory | Where are decisions and failures recorded? |  |

## Minimum Setup

1. Fill `context/project-brief.md`.
2. Pick one skill from `skills/`.
3. Define one human gate from `harness/human-gates.md`.
4. Use one loop from `loops/`.
5. Record outcomes in `context/decision-log.md`.

## Quality Checks

- The harness starts from a real task.
- It has an output contract.
- It has at least one human gate.
- It records failures and decisions.

## Human Gates

The PM must approve roadmap priority, user commitments, sensitive data use, publication, and launch.
